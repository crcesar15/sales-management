# Frontend — Product Variants & Options

## Pages & Components

```
resources/js/Pages/Products/
├── Show.vue                         # Product detail — hosts options + variants panels
└── Partials/
    ├── OptionsPanel.vue             # Options list + add-value inline; triggers auto-generate
    ├── VariantsPanel.vue            # Variant DataTable with inline edit
    ├── AutoGenerateDialog.vue       # Option/value builder + combination preview
    └── ManualVariantDialog.vue      # Single variant add form
```

## PrimeVue Components Used

| Component | Usage |
|---|---|
| `DataTable` + `Column` | Variant list (identifier, price, stock, status, actions) |
| `InputText` | Variant identifier, option name/value inputs |
| `InputNumber` | Price (mode="currency") and stock fields |
| `Dropdown` | Variant status selector |
| `Tag` | Variant status badge |
| `Button` | Add option, add value, generate, manual add, edit, delete |
| `Dialog` | AutoGenerateDialog, ManualVariantDialog |
| `Chip` / `Tag` | Display option values per variant cell |
| `Toast` | Feedback |

## Key Inertia Patterns

**Product show page receives full options + variants in props:**
```ts
defineProps<{
  product: ProductDetailResource  // includes options[] with values[], variants[]
}>()
```

**Auto-generate — local form state (no Inertia form until submit):**
```ts
// AutoGenerateDialog.vue
const optionRows = ref([{ name: '', values: [''] }])
const combinationCount = computed(() =>
  optionRows.value.reduce((acc, row) => acc * Math.max(row.values.filter(Boolean).length, 1), 1)
)

const form = useForm({ options: optionRows })
form.post(route('products.variants.generate', product.id), {
  onSuccess: () => { dialogVisible.value = false }
})
```

**Manual variant form:**
```ts
const form = useForm({
  identifier: '', price: 0, stock: 0, status: 'active', option_value_ids: []
})
form.post(route('products.variants.store', product.id))
```

**Inline variant edit — PUT:**
```ts
const editForm = useForm({ identifier: v.identifier, price: v.price, stock: v.stock, status: v.status })
editForm.put(route('products.variants.update', [product.id, variant.id]))
```

## OptionsPanel Layout

```
[+ Add Option]
  Color          [+ Add Value]
    • Red  [×]
    • Blue [×]
  Size           [+ Add Value]
    • S [×]  • M [×]  • L [×]
[Generate Variants from Options]  (shown only if no variants exist)
```

- "×" on a value sends `DELETE` — server returns 422 if in use
- "Generate" button opens `AutoGenerateDialog`

## VariantsPanel Layout

DataTable columns: Identifier | Options (chips) | Price | Stock | Status | Actions

- Price displayed with currency formatting via `InputNumber` in edit mode
- Stock is **read-only display** — only editable when creating, not after (Phase 3 controls it)
- Status dropdown: active / inactive / archived

## TypeScript Types
```ts
interface ProductOptionResource {
  id: number; name: string
  values: { id: number; value: string }[]
}
interface ProductVariantResource {
  id: number; identifier: string | null
  price: string; stock: number; status: string
  option_values: { id: number; value: string }[]
}
```

## Gotchas
- Combination count preview must update reactively as user adds/removes values in `AutoGenerateDialog`
- Disable "Generate" button if any option has zero values or if variants already exist
- Stock column in the variant table should show the current value but be read-only (tooltip: "Managed by inventory")
